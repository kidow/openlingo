# ChatGPT Apps 출시를 위한 MCP 서버 구축 계획

## 1. 문서 목적

이 문서는 OpenLingo를 ChatGPT Apps로 확장해, 학습 대화 중 즉시 사용할 수 있는:

- 문자별 손글씨 연습 **Canvas UI**
- 입력받은 문장을 음성으로 재생하는 **TTS Audio 기능**

을 제공하기 위한 MCP 서버 및 제품/운영 계획을 정리한다.

---

## 2. 제품 비전과 사용자 경험

### 2.1 목표 경험

사용자가 ChatGPT로 언어를 공부하는 중, 아래 흐름을 자연스럽게 반복한다.

1. ChatGPT가 연습 문장 또는 단어를 제시한다.
2. 사용자는 앱 내 Canvas에서 문자를 따라 쓰거나 받아쓰기 형태로 직접 쓴다.
3. 앱은 stroke 데이터를 기반으로 유사도/피드백을 제공한다.
4. 같은 문장을 TTS로 생성해 발음을 반복 청취한다.
5. 사용자는 다시 쓰고, 다시 듣고, 점진적으로 정확도를 높인다.

### 2.2 핵심 가치

- 대화형 학습(LLM) + 필기 훈련(Canvas) + 청취 훈련(TTS)을 한 흐름으로 결합
- 문자 단위 반복 학습에 강한 OpenLingo 강점을 ChatGPT 학습 맥락에 연결
- 개인 학습자 기준으로도 즉시 체감 가능한 실전형 학습 루프 제공

---

## 3. 범위 정의

### 3.1 1차 출시(MVP) 범위

- ChatGPT Apps에서 실행 가능한 연습 캔버스 컴포넌트
- 문자/단어/짧은 문장 템플릿 선택 및 쓰기 입력
- stroke 수집 및 기본 점수화(기존 유사도 로직 재사용)
- 입력 문장 기반 TTS 생성 및 재생
- 세션 단위 로컬 저장(또는 경량 서버 저장)

### 3.2 제외 범위(초기)

- 계정 기반 멀티디바이스 동기화
- 고급 AI 필적 교정(획순 강제, 자세 교정 등)
- 긴 음성 파일 편집/가공 파이프라인
- 대규모 B2B 운영을 가정한 고가용성 설계

---

## 4. 시스템 아키텍처

### 4.1 구성요소

1. **ChatGPT Apps UI (Frontend)**
   - Canvas 기반 필기 입력
   - 연습 문장 표시, 점수/피드백 표시
   - TTS 재생 컨트롤(재생/정지/반복)

2. **MCP Server (Tool Orchestrator)**
   - ChatGPT가 호출할 도구(tool) 제공
   - 연습 항목 생성, 평가 요청, TTS 요청 라우팅
   - 세션 메타데이터 처리

3. **Application API (Domain Backend)**
   - stroke/세션 저장
   - 유사도 계산 API (혹은 프론트 계산 결과 검증)
   - TTS 생성 요청 및 음원 URL 반환

4. **Storage/CDN**
   - 오디오 파일 저장(단기 캐시 + 만료 정책)
   - 정적 자산/결과 전달

5. **Observability**
   - 요청량, 실패율, 지연시간, TTS 사용량 모니터링

### 4.2 데이터 흐름(요약)

1. 사용자 문장 입력 → ChatGPT Apps UI
2. UI가 MCP tool 호출(`generate_tts`, `save_attempt` 등)
3. MCP가 Application API/TTS Provider 호출
4. 음원 URL/평가 결과를 UI로 반환
5. UI에서 오디오 재생 + Canvas 피드백 업데이트

---

## 5. MCP 도구 설계

### 5.1 권장 Tool 목록

- `get_practice_prompt`
  - 입력: 언어, 난이도, 길이
  - 출력: 연습 문장/단어, 문자 가이드 메타

- `score_handwriting_attempt`
  - 입력: 템플릿 ID, stroke 데이터(또는 요약), 옵션
  - 출력: 점수(0~100), 피드백 레이블, 개선 포인트

- `generate_tts_audio`
  - 입력: 텍스트, 언어 코드, 보이스, 속도
  - 출력: 오디오 URL, duration, 캐시 키

- `save_practice_session`
  - 입력: 문장, 점수, 시도 횟수, 선택적 오디오 메타
  - 출력: 세션 ID, 저장 결과

- `get_session_summary`
  - 입력: 기간/세션 ID
  - 출력: 평균 점수, 취약 문자, 추천 다음 연습

### 5.2 응답 표준

- 모든 Tool 응답에 `request_id`, `timestamp`, `version` 포함
- 에러는 사용자 표시용 메시지 + 개발자 디버그 코드 분리
- TTS URL은 만료 시간(`expires_at`)을 명시

## 6. Canvas 학습 모듈 구현 계획

### 6.1 UI/입력

- Pointer Events 기반 stroke 수집(`pointerdown/move/up/cancel`)
- 손가락 입력 허용 on/off 토글
- Undo/Clear/Replay 버튼
- 문자 가이드 오버레이(SVG) + 사용자 stroke 레이어(Canvas)

### 6.2 평가

- 기존 유사도 계산 로직 재사용
- 입력 종료 debounce 후 자동 점수화
- 점수에 따른 3단계 피드백(좋음/보통/연습 필요)

### 6.3 UX 최적화

- iPad/펜 환경 우선 최적화, 안드로이드 태블릿 호환
- 입력 지연 최소화(60fps 목표)
- 문장 길이에 따라 자동 줄바꿈/캔버스 분할

---

## 7. TTS 오디오 모듈 구현 계획

### 7.1 기능 요구사항

- 사용자 입력 문장 즉시 TTS 생성
- 언어별 voice 프리셋 제공(예: ko-KR, en-US, ja-JP)
- 속도(0.8x~1.2x), 반복 재생 옵션

### 7.2 처리 전략

- 동일 텍스트/보이스/속도 조합은 캐시 재사용
- 짧은 문장은 실시간 생성, 긴 문장은 비동기 처리 + 로딩 표시
- URL 만료/재발급 전략 운영

### 7.3 품질 원칙

- 발음 정확도 우선, 과한 음색 커스터마이징은 후순위
- 학습용 문장은 명료성 중심 보이스 기본값 제공

---

## 8. 운영비/인프라 계획 (개인 사용자 기준)

### 8.1 비용 구성

- 고정비: 서버/호스팅, 로그/모니터링 기본 요금
- 변동비: LLM 호출, TTS 호출, 저장소 트래픽

### 8.2 비용 최소화 전략

- TTS 캐시 적극 적용(같은 문장 재생성 방지)
- 긴 문장 제한 또는 분할 처리
- 세션/이력 보관기간 제한으로 저장비 절감
- 월 사용량 상한(하드 리밋) 설정

### 8.3 운영 정책

- 일 단위 예산 알림
- 에러율/지연시간 임계치 알람
- 비정상 호출 패턴 자동 차단(기본 rate limit)

---

## 9. 보안/개인정보/안정성

- 최소 수집 원칙(학습에 불필요한 개인식별정보 저장 금지)
- 전송 구간 암호화(HTTPS)
- 세션 기반 접근 제어
- 오디오 URL 서명 및 만료
- 장애 대비: TTS 실패 시 텍스트 학습 모드로 자동 폴백

---

## 10. 단계별 실행 로드맵

### Phase 1: 기술 검증 (1~2주)

- ChatGPT Apps 컴포넌트에서 Canvas 상호작용 검증
- MCP `generate_tts_audio` 단일 도구 연결
- 최소 학습 루프(입력→듣기→쓰기) 데모

### Phase 2: MVP 구현 (2~4주)

- 핵심 MCP tool 세트 구현
- Canvas 점수화 + 세션 저장
- TTS 캐시/만료 정책 적용

### Phase 3: 안정화/출시 (1~2주)

- 성능 측정(응답속도, 렌더링 부하)
- 운영 대시보드/알람 연결
- 제한적 사용자 테스트 후 ChatGPT Apps 공개

### Phase 4: 고도화 (출시 후)

- 약점 문자 자동 추천
- 받아쓰기 모드(오디오 우선 제시 후 필기)
- 언어별 학습 코스 패키지 확장

---

## 11. 성공 지표(KPI)

- Canvas 완주율(연습 시작 대비 제출 비율)
- 평균 점수 상승률(세션 1회차 대비 N회차)
- TTS 재생 재시도율(청취 집중도 지표)
- 일/주간 재방문율
- 사용자당 평균 API 비용
