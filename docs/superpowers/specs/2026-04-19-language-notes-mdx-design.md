# 언어별 자유 MDX 노트 설계

Date: 2026-04-19
Status: Draft

## 요약

`/notes/[lang]` 아래에 언어별 MDX 문서를 1개씩 두고, 각 문서는 해당 언어에서 사용자가 그때그때 배운 내용을 자유롭게 누적하는 개인 학습 노트로 사용한다. 문서 내부 구조는 강제하지 않으며, 각 언어는 서로 다른 주제, 순서, 밀도를 가질 수 있다. 공통화는 문서를 렌더링하는 외곽 레이아웃, frontmatter 메타데이터, 자동 목차, MDX 컴포넌트 주입 정도까지만 적용한다.

## 배경

OpenLingo에는 이미 언어별 연습 화면과 템플릿 기반 콘텐츠 구조가 있다. 이번 기능은 그와 별개로, 학습자가 언어별로 새로 배운 것들을 즉시 적어두고 AI가 계속 참조할 수 있는 자유 노트를 제공하는 것이 목적이다.

이 노트는 교재나 커리큘럼이 아니다. 문서가 깔끔하게 정돈되어 있을 필요는 없고, 메모가 뒤섞여 있어도 된다. 중요한 것은 언어별로 단일 진입점이 있고, 내용을 계속 덧붙일 수 있으며, 필요할 때 React 컴포넌트를 섞을 수 있다는 점이다.

## 목표

- 언어별 MDX 파일 1개를 제공한다.
- 각 언어 파일은 자유 형식으로 내용을 누적한다.
- 앱은 `/notes/[lang]`에서 해당 문서를 렌더링한다.
- frontmatter를 사용해 제목, 설명, SEO 메타를 관리한다.
- 문서 내 헤딩을 기반으로 자동 목차를 제공한다.
- MDX 안에서 발음 버튼, 플래시카드, 예문 카드 같은 React 컴포넌트를 사용할 수 있게 한다.

## 비목표

- 문서 구조를 언어 간 동일하게 강제하지 않는다.
- 시간순 학습 로그로 정리하지 않는다.
- 앱 안에서 문서를 편집하지 않는다.
- 협업형 CMS나 서버 저장소를 도입하지 않는다.
- 교정, 채점, 학습 진도 관리 시스템으로 확장하지 않는다.

## 권장 구조

### 콘텐츠 저장

언어별 문서는 단일 MDX 파일로 둔다.

- `src/content/notes/ja.mdx`
- `src/content/notes/en.mdx`
- `src/content/notes/zh-hans.mdx`

파일 하나가 해당 언어의 전체 노트 역할을 한다. 언어마다 문서 내용과 섹션 제목, 섹션 순서는 서로 달라도 된다.

### 라우팅

실제 노출 경로는 `/notes/[lang]`이다.

- `/notes/ja`
- `/notes/en`
- `/notes/zh-hans`

라우트는 해당 언어의 MDX 파일을 찾아 읽고 렌더링한다. 언어별 파일이 없으면 404를 반환한다.

### 공통 레이아웃

공통 UI는 다음 정도만 담당한다.

- 문서 제목
- 간단한 메타 정보
- 자동 목차
- 본문 컨테이너
- SEO 메타 생성

본문 내용은 언어별 문서가 전부 결정한다. 공통 레이아웃은 본문 구조를 해석하거나 정리하지 않는다.

## Frontmatter 규약

각 MDX 파일은 frontmatter를 가진다. 필수/권장 필드는 다음과 같다.

### 필수

- `title`: 페이지 제목
- `description`: 짧은 문서 설명
- `lang`: 언어 코드
- `updatedAt`: 마지막 갱신일

### 권장

- `seoTitle`: 검색 결과용 제목
- `seoDescription`: 검색 결과 및 공유용 설명
- `ogImage`: 공유 카드용 이미지 경로 또는 식별자

이 메타데이터는 페이지 상단 정보와 Next.js metadata 생성에 사용한다.

## 렌더링 규칙

### 본문

MDX 본문은 자유 형식이다. 섹션 수, 순서, 제목 스타일을 강제하지 않는다. 사용자는 새로 배운 표현, 문법, 예문, 메모, 비교 노트, 암기 팁 등을 뒤섞어서 기록할 수 있다.

### React 컴포넌트

MDX 안에서는 다음과 같은 컴포넌트를 필요한 만큼 포함할 수 있다.

- 발음 재생 버튼
- 플래시카드
- 예문 카드
- 주의사항 박스
- 보조 설명용 인라인 UI

문서별로 실제로 쓰는 컴포넌트는 달라질 수 있다. 공통 레이아웃은 컴포넌트가 어떤 순서로 나오든 문제없이 렌더링해야 한다.

### 목차

목차는 문서 내 헤딩을 기준으로 자동 생성한다.

- `h2`, `h3` 등의 헤딩을 수집한다.
- 내용의 논리 구조를 바꾸지 않는다.
- 목차는 내비게이션용일 뿐, 문서 편집 규칙을 강제하지 않는다.

## SEO 규칙

각 언어 페이지는 문서별 메타데이터를 기반으로 SEO를 생성한다.

- `title` 또는 `seoTitle`을 페이지 제목으로 사용한다.
- `description` 또는 `seoDescription`을 메타 설명으로 사용한다.
- canonical은 `/notes/[lang]` 구조를 따른다.
- OG 메타는 `ogImage`가 있으면 사용한다.

문서는 검색엔진에도 개별 학습 노트로 노출될 수 있어야 한다.

## AI 갱신 규칙

AI는 각 언어 파일을 지속적으로 참조하고 업데이트한다.

- 새로운 학습 내용은 해당 언어 MDX 파일에 바로 추가한다.
- 기존 섹션을 억지로 재정렬하지 않는다.
- 문서가 뒤죽박죽이어도 구조를 다시 강제하지 않는다.
- 의미상 가까운 위치가 있으면 거기에 덧붙이고, 그렇지 않으면 문서 하단의 새 섹션으로 추가한다.
- 섹션 제목은 사용자가 정한 표현을 우선 존중한다.

이 규칙의 목적은 문서를 “정리 잘 된 아카이브”가 아니라 “계속 쌓이는 살아 있는 노트”로 유지하는 것이다.

## 사용자 경험

- 사용자는 `/notes/[lang]`에서 해당 언어의 전체 노트를 본다.
- 페이지는 읽기 전용이다.
- 문서가 길어지면 목차로 빠르게 이동할 수 있다.
- 노트 안에서 바로 발음이나 예문 보조 UI를 볼 수 있다.
- 언어별 노트는 서로 다른 성격을 가질 수 있으므로, 사용자는 언어마다 다른 방식으로 내용을 쌓아도 된다.

## 구현 기준

이 설계가 만족되려면 다음 조건을 충족해야 한다.

- 언어별 MDX 파일이 1개씩 존재한다.
- `/notes/[lang]`로 해당 문서가 렌더링된다.
- 문서 frontmatter가 SEO와 페이지 메타 생성에 연결된다.
- 문서 안의 헤딩을 목차로 노출한다.
- MDX 안에서 React 컴포넌트를 사용할 수 있다.
- 문서 본문 구조는 언어별로 서로 달라도 된다.
