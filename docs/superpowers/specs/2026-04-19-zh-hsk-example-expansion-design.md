# 중국어 간체 HSK 예시 단어 확장 설계

Date: 2026-04-19
Status: Draft

## 요약

HSK 1급 단어 목록을 중국어 간체 예시 단어의 원천 데이터로 사용하되, 노출 단위는 단어가 아니라 문자 템플릿으로 유지한다. 즉, `中国` 같은 항목은 하나의 단어로 저장하더라도 `中`와 `国` 템플릿에 모두 예시로 연결한다. 번체에는 HSK 예시 단어를 적용하지 않는다.

## 문제 정의

현재 중국어 예시 단어는 일부 기초 한자 템플릿에만 수동으로 붙어 있다. 이 방식은 작은 세트에는 충분하지만, HSK 단어 150개처럼 규모가 커지면 다음 문제가 생긴다.

- 단어 단위로만 관리하면 실제 연습 단위인 문자 템플릿과 분리된다.
- 여러 문자가 들어 있는 단어를 어디에 붙일지 애매해진다.
- 새 문자 템플릿을 추가할 때 기존 단어 예시를 다시 찾고 붙여야 한다.

이 작업의 핵심은 HSK 목록을 체계적으로 흡수하면서도, 화면에서는 계속 문자 연습 중심 구조를 유지하는 것이다.

## 목표

- HSK 1급 단어를 중국어 예시 데이터로 흡수한다.
- 예시 단어는 문자 템플릿 기준으로 붙인다.
- 다문자 단어는 포함된 각 문자 템플릿에 모두 연결한다.
- 간체 예시 단어를 문자 템플릿 기준으로 붙인다.
- 기존 `ExampleWordsSheet`와 중국어 전용 시트 구조를 유지한다.

## 비목표

- HSK 단어를 별도의 새 UI 섹션으로 보여주지 않는다.
- 단어 단위 학습 화면을 새로 만들지 않는다.
- 중국어 예시 단어 전용 독립 페이지를 추가하지 않는다.
- 현재 쓰기 연습 캔버스 구조를 바꾸지 않는다.

## 권장 접근

`문자 우선 + HSK fan-out 매핑` 방식을 사용한다.

이 접근을 추천하는 이유는 다음과 같다.

- 현재 중국어 연습은 문자 템플릿이 중심이다.
- 예시 단어도 같은 중심축을 따라가야 사용자가 학습 맥락을 잃지 않는다.
- HSK 목록이 커져도 데이터 구조가 단순하게 유지된다.
- 번체 예시는 이번 범위에 포함하지 않는다.

## 데이터 모델

### 원본 단어 목록

HSK 파일은 원본 레퍼런스로 취급한다. 각 행은 다음 정보를 가진다.

- 간체 단어
- 병음
- 의미

이 원본을 그대로 보존한 뒤, 파이프라인에서 문자 단위 연결에 사용한다.

### 문자 템플릿 키

중국어 HSK 예시 단어는 `zh-hans-<suffix>` 형태의 템플릿 키에만 연결한다.

예:

- `中国` 같은 항목은 현재 존재하는 `中` 및 `国` 템플릿에 모두 연결한다
- 여러 문자가 들어 있는 항목은 각 문자의 실제 템플릿이 있는 경우에만 fan-out 한다
- `火车站` 같은 항목은 `火`, `车`, `站` 템플릿이 모두 존재할 때 전부 연결한다

핵심은 “단어가 어디에 속하느냐”가 아니라 “단어 안의 각 문자가 어떤 템플릿을 갖고 있느냐”이다.

## 동작 방식

### 1. HSK 항목 수집

HSK 1급 목록을 읽어 들여 단어 레코드로 정리한다. 각 레코드는 원문을 유지한다.

### 2. 문자 분해

각 단어를 문자 단위로 분해한다. 예를 들어 `中国`은 `中`와 `国`으로 분해된다.

### 3. 템플릿 매칭

분해된 문자들에 대해, 현재 존재하는 중국어 템플릿 키를 찾는다.

- 매칭되는 문자 템플릿이 있으면 예시를 붙인다.
- 매칭되지 않는 문자는 누락 리스트에 남긴다.

### 4. fan-out 연결

한 단어는 매칭된 모든 문자 템플릿에 동일 예시로 연결한다.

이 방식 때문에 `中国`은 `中`의 예시이면서 동시에 `国`의 예시가 된다.

### 5. 시트 렌더링

중국어 간체 예시 시트는 계속 `templateId -> example words` 맵만 읽는다.

- `selectedTemplateId`에 해당하는 예시 목록만 보여준다.
- 데이터가 없으면 기존 unavailable 메시지를 보여준다.
- 음성 선택은 중국어 전용 프로필을 유지한다.

## 컴포넌트 경계

### `src/data/templates/zh/examples.ts`

중국어 간체 예시 단어를 템플릿 키별로 제공하는 데이터 레이어다.

이 레이어는 다음 역할을 맡는다.

- HSK 원본 데이터 정리
- 간체 기본 예시 유지
- 템플릿 키별 export 생성

### `src/components/practice/chinese-example-sheet.tsx`

중국어 예시 시트는 기존대로 얇은 래퍼로 유지한다.

- 선택된 템플릿의 예시를 읽는다.
- 간체 전용 HSK 예시를 읽는다.
- `ExampleWordsSheet`에 필요한 props만 전달한다.

### `src/lib/speech-synthesis.ts`

중국어 음성 목록은 다른 언어 팩과 같은 프로필 방식으로 유지한다.

- 간체 프로필은 순서 있는 allowlist로 유지
- 표시 가능한 음성은 프로필 기반으로 정렬
- 기본 음성도 프로필 순서를 따른다

## 확장 규칙

### 문자 추가 규칙

새로운 HSK 단어가 들어오면 다음 순서로 처리한다.

1. 단어에 포함된 문자를 확인한다.
2. 해당 문자의 템플릿이 있는지 확인한다.
3. 없으면 문자 템플릿을 먼저 추가한다.
4. 그 다음 HSK 단어를 해당 문자 템플릿들에 모두 연결한다.

### 중복 연결 규칙

다문자 단어는 중복 연결을 허용한다.

예:

- `中国`는 `中`와 `国`에 모두 붙는다.
- 여러 문자가 들어 있는 HSK 항목은 포함된 각 문자 템플릿에 모두 붙는다.

중복을 제거하지 않는 이유는 각 문자가 단어 안에서 반복 노출되는 것이 연습에 더 유리하기 때문이다.

## 오류 및 누락 처리

- 어떤 단어의 문자 중 일부만 템플릿이 있어도, 연결 가능한 문자에는 예시를 붙인다.
- 완전히 매칭되는 템플릿이 없으면 그 단어는 누락 목록에 남긴다.
- 예시가 없는 템플릿은 기존 unavailable 메시지를 유지한다.

## 사용자 경험

- 사용자는 기존과 같은 예시 단어 버튼으로 중국어 예시를 연다.
- 선택된 템플릿에 연결된 HSK 단어 예시만 보인다.
- 한 단어가 여러 문자에 붙어 있어도, 각각의 문자 학습 맥락 안에서만 읽힌다.
- 번체 화면에는 HSK 예시가 나오지 않는다.

## 검증 전략

### 데이터 검증

- HSK 원본에서 추출한 단어 수와 연결된 템플릿 수를 확인한다.
- 누락된 문자 목록을 확인해 아직 없는 템플릿을 식별한다.
- 간체 예시 키 수가 예상대로 생성되는지 확인한다.

### 수동 확인

- `中`, `国`, `学`, `车`, `站` 같은 대표 문자에서 예시가 보이는지 확인한다.
- 다문자 단어가 여러 문자 템플릿에 모두 표시되는지 확인한다.
- 중국어 팩에서 예시 단어 버튼이 계속 노출되는지 확인한다.

### 코드 검증

- 데이터 파일의 템플릿 키가 실제 존재하는 중국어 템플릿 ID와 일치하는지 확인한다.
- 중국어 시트가 비어 있지 않은 선택 템플릿에서 예시를 정상 렌더링하는지 확인한다.

## 구현 순서 메모

1. HSK 단어 목록을 문자 단위로 파싱할 수 있는 원본 레이어를 만든다.
2. 기존 중국어 간체 예시 맵에 HSK 단어를 fan-out 하는 변환 레이어를 넣는다.
3. 누락 문자 목록을 보고 필요한 중국어 간체 템플릿을 추가한다.
4. 중국어 예시 시트와 음성 프로필이 계속 정상 노출되는지 확인한다.
